## Findings Requiring Action

### High Priority

**H-1: `SessionLogUpdate.status` has no enum constraint**  
`status` is `Optional[str]` — any string passes Pydantic and gets `setattr`'d onto the ORM model. An attacker could set `status="hacked"`. Fix: Add `Literal["planned", "in_progress", "completed", "skipped"]` constraint. Related: `SessionLogCreate.session_type` has the same gap.

**H-2: `_apply_action` complete_session can overwrite `id` field**  
The `setattr` loop has `'session_log_id'` in the exclusion list but not `'id'`. Since `hasattr(SessionLog, 'id')` is True, a crafted payload with `{"id": "new-uuid"}` would change the session's primary key. Fix: Add an explicit allowlist of updatable fields (e.g., `status`, `overall_rpe`, `completed_at`, `notes`).

### Medium Priority

**M-1: `httpx.AsyncClient` is created per-request**  
`AIClient._http_post` uses `async with httpx.AsyncClient(...)` inside the method, creating and discarding a connection on every AI call. Connection pooling would reduce latency. Fix: Initialize a shared client in `AIClient.__init__` and reuse it.

**M-2: `coaching_engine.chat` stores user message before AI call**  
If AI generation fails after the user message is persisted, the user message row exists without a matching assistant message. The fallback text prevents "true orphans" (a fallback assistant message is always stored), but the ordering still means a brief inconsistency window exists.

**M-3: `datetime.utcnow()` is deprecated**  
Python 3.12+ recommends `datetime.now(timezone.utc)`. Appears in `sessions.py`, `offline.py`, and `coaching_engine.py`. Low urgency but should be migrated.
